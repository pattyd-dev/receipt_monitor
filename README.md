# Infrastructure Monitoring with Grafana on AWS

# Overview
This project deploys a self-hosted Grafana instance on a dedicated EC2 instance to provide centralized, real-time observability across an existing AWS infrastructure. Metrics from compute, container, and database layers are consolidated into a single monitoring platform, enabling proactive alerting and rapid incident response without any manual intervention.
# Architecture
Grafana runs on a standalone EC2 instance and integrates with AWS CloudWatch as its primary data source. Application-tier EC2 instances are monitored for CPU, memory, disk I/O, and network throughput. ECS containers are tracked at the cluster and task level, exposing CPU and memory reservation, task counts, and service utilization. The RDS instance is monitored for query latency, connection counts, read/write IOPS, freeable memory, and storage capacity. All metrics are surfaced through unified Grafana dashboards, allowing operators to correlate events across all three layers from a single interface.
# Security
Security is enforced through tightly scoped security groups and IAM policies, adhering strictly to the principle of least privilege. Each security group permits only the minimum required ingress and egress traffic — the Grafana instance accepts inbound traffic solely on the UI port from authorized IP ranges, with all other traffic denied by default. The instance is assigned an IAM role granting read-only CloudWatch access limited to only the actions required for metric retrieval, with no write permissions or access to unrelated services. Proper subnetting isolates the monitoring instance from application and database subnets, ensuring the monitoring plane remains fully decoupled from production workloads.
